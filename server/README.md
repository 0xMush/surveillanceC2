# C2 Panel

Secure, multi-file MVC architecture C2 panel with SQLite backend.

## Requirements
- PHP >= 8.1 (7.4 minimum)
- pdo_sqlite extension
- fileinfo extension

## Quick Start
1. Navigate to `setup.php` in your browser
2. Admin credentials are generated and displayed
3. **Save the credentials** - the page self-destructs
4. Login at `index.php`
5. Delete `setup.php` (already removed automatically)
6. Delete `README.md` in production

## Architecture
```
server/
├── index.php          # Frontend shell (served to panel admins)
├── api.php            # Single API entry point (JSON only)
├── setup.php          # First-run setup (self-destructs)
├── .env               # BEACON_SECRET only (generated by setup)
├── .htaccess          # URL rewriting + security headers
├── assets/
│   ├── panel.css      # Dashboard styles
│   └── panel.js       # Frontend logic (ApiClient + UI)
├── includes/
│   ├── config.php     # Bootstrap + env loading
│   ├── db.php         # PDO SQLite wrapper + schema
│   ├── auth.php       # Session, CSRF, rate limiting
│   ├── helpers.php    # Utility functions
│   └── router.php     # Request routing
├── handlers/
│   ├── auth.php       # Login/logout
│   ├── beacon.php     # Beacon check-in + task polling
│   ├── task.php       # Task CRUD
│   ├── result.php     # Result submission + auto-cache
│   ├── file.php       # File upload/download
│   ├── media.php      # Media upload/serve
│   ├── device.php     # Device management
│   ├── filebrowser.php # Browse cache + remote file ops
│   └── persons.php    # Person/CTOS management
├── data/
│   ├── c2.sqlite      # SQLite database (auto-created)
│   └── .htaccess      # Deny all
└── uploads/
    ├── media/         # Screenshots, camera captures
    └── persons/       # Person profile photos
```

## Security
- **BEACON_SECRET** in `.env` (64+ chars, auto-generated)
- Panel admin credentials stored in database (argon2id/bcrypt)
- CSRF tokens per session on all non-GET panel actions
- Login rate limiting: 5 attempts per 15 minutes per IP
- Beacon endpoints require `Authorization: Bearer {BEACON_SECRET}`
- All database queries use PDO prepared statements
- File uploads validated with `finfo_file()` MIME detection
- Uploads stored outside web root
- Protected directories blocked via `.htaccess`
- `includes/` and `handlers/` directories blocked from web access

## Beacon Communication
Beacons authenticate via HTTP header:
```
Authorization: Bearer your-beacon-secret-here
```

### Beacon Check-in POST /api.php?action=beacon
```json
{
  "uuid": "unique-beacon-id",
  "hostname": "client-hostname",
  "os": "Windows 10",
  "username": "user",
  "privilege": "user",
  "pid": 1234,
  "ip": "192.168.1.100"
}
```

### Result Submission POST /api.php?action=result
```json
{
  "task_id": 1,
  "beacon_uuid": "unique-beacon-id",
  "output": "command output here",
  "status": "completed"
}
```
